     1→# MCP Server Charm for Juju — Plan
     2→
     3→## Overview
     4→
     5→A **subordinate machine charm** that deploys an MCP (Model Context Protocol) server
     6→on the same machine as a principal charm. The principal declares tools, prompts,
     7→and resources via Juju integration data. The MCP server charm aggregates these
     8→declarations and serves them over streamable HTTP. Ingress is provided via Traefik.
     9→
    10→Clients (LLM agents, Claude Code, etc.) connect to the MCP server endpoint and
    11→can discover and invoke the tools/prompts/resources declared by any integrated
    12→principal charm — without the principal needing to implement MCP itself.
    13→
    14→## Architecture
    15→
    16→```
    17→                  ┌─────────────────────────────────────────────┐
    18→                  │              Machine (Juju unit)            │
    19→                  │                                             │
    20→  Client ──► Traefik ──► MCP Server (subordinate charm)        │
    21→                  │            │                                │
    22→                  │            │ exec / shell commands          │
    23→                  │            ▼                                │
    24→                  │       Principal workload                    │
    25→                  │       (e.g. PostgreSQL, app, etc.)          │
    26→                  └─────────────────────────────────────────────┘
    27→```
    28→
    29→- The MCP server runs as a systemd service on the machine.
    30→- On `call_tool`, it executes the declared handler (shell command, script, or
    31→  HTTP call to localhost).

    33→  directly.

   139→
   140→### `exec` — Run a shell command
   141→
   142→```json
   143→{
   144→  "type": "exec",
