# Adding Tracing to a Charm with ops[tracing]

## Dependencies
Add `ops[tracing]` to your `pyproject.toml` or `requirements.txt`.

## Configuration in charmcraft.yaml
Declare tracing relations:
```yaml
requires:
  charm-tracing:
    interface: tracing
    limit: 1
    optional: true
  receive-ca-cert:
    interface: certificate_transfer
    limit: 1
    optional: true
```

## Charm Implementation

**Imports:**
```python
import ops.tracing
import opentelemetry.trace  # For custom instrumentation
```

**Initialization in __init__ method:**
```python
class MyCharm(ops.CharmBase):
    def __init__(self, framework: ops.Framework):
        super().__init__(framework)
        self.tracing = ops.tracing.Tracing(
            self,
            tracing_relation_name='charm-tracing',
            ca_relation_name='receive-ca-cert',
        )
```

## Custom Instrumentation

Create a tracer and use context managers for spans:
```python
tracer = opentelemetry.trace.get_tracer(__name__)

with tracer.start_as_current_span('span-name') as span:
    span.add_event('event-name', {'attribute': value})
```

**Note:** No decorators are used; "prefer using the OpenTelemetry `start_as_current_span` primitive as a context manager" for better ergonomics.

## Integration
Deploy and integrate with a tracing provider:
```bash
juju deploy my-charm
juju integrate my-charm tempo
```
