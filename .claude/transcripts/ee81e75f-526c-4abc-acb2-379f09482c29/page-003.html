## Phase 6: Observability / COS Integration -- Implementation Plan

### Architectural Context and Key Constraint

The MCP server charm is a **subordinate** charm. The grafana-agent charm is **also a subordinate**. Juju does not support subordinate-to-subordinate relations with `scope: container`. However, the established pattern (used by `mysql-router-operator` and others) is:

1. The subordinate charm **provides** a `cos-agent` relation (interface `cos_agent`, **without** `scope: container`).
2. The `grafana-agent` subordinate is deployed by relating it to the **principal** application (via `juju-info`), which places it on the same machine.
3. Then `grafana-agent` is related to the subordinate's `cos-agent` endpoint as a regular (non-subordinate) relation.
4. The `COSAgentProvider` library bundles metrics endpoints, Grafana dashboards, alert rules, log slots, and tracing protocol declarations -- grafana-agent reads all of this and forwards telemetry to COS Lite (Prometheus, Loki, Grafana, Tempo) running in a k8s model via cross-model relations.

This means the MCP server charm does **not** need direct relations to Prometheus, Loki, or Grafana. Everything flows through the single `cos-agent` relation to `grafana-agent`.

### Overview of Items

| Roadmap Item | Approach | Feasibility |
|---|---|---|
| Tracing (charm) | `@trace_charm` decorator via `charm_tracing` library, relayed through `COSAgentProvider(tracing_protocols=["otlp_http"])` | Fully feasible |
| Tracing (workload) | OpenTelemetry instrumentation in the ASGI app, export to grafana-agent's OTLP endpoint | Feasible but heavier; defer to later |
| Metrics endpoint | Add `/metrics` endpoint to workload using `prometheus_client`, declare in `COSAgentProvider(metrics_endpoints=[...])` | Fully feasible |

    async def dispatch(self, request: Request, call_next: Any) -> Response:
        method = request.method
        path = request.url.path
        ACTIVE_CONNECTIONS.inc()
        start = time.monotonic()
        try:
            response = await call_next(request)
            REQUEST_COUNT.labels(method=method, endpoint=path, status=response.status_code).inc()
            return response
        except Exception:
